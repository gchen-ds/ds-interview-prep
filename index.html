Day 2: Probability Distributions

Discrete & Continuous distributions — the building blocks of every data science interview at FAANG companies.

Core Distributions

Master the PMF/PDF, expected value, and variance for each. These formulas come up directly in interviews.

🎯
Binomial Distribution
n independent trials, each with probability p of success
DISCRETE

Models the number of successes in n independent Bernoulli trials. Think: "Out of n attempts, how many succeed?"

PMF P(X = k) = C(n,k) · pk · (1−p)n−k   for k = 0, 1, …, n
Expected Value E[X] = n · p
Variance Var(X) = n · p · (1 − p)
💡
Key intuition: Variance is maximized when p = 0.5 (most uncertainty). As p → 0 or p → 1, variance shrinks because outcomes become predictable.

When to use it

  • Fixed number of trials (n is known)
  • Each trial is independent with same probability p
  • You care about the count of successes
⚡
Poisson Distribution
Count of events in a fixed interval, rate = λ
DISCRETE

Models the number of events occurring in a fixed interval of time/space when events happen at a constant average rate λ and independently.

PMF P(X = k) = e−λ · λk / k!   for k = 0, 1, 2, …
Expected Value E[X] = λ
Variance Var(X) = λ
💡
Key insight: E[X] = Var(X) = λ. This is the signature property of Poisson. If your data's mean ≈ variance, it's likely Poisson. If variance >> mean, consider Negative Binomial (overdispersion).

When to use it

  • Events are rare relative to opportunities
  • Events occur independently at a constant rate
  • You're counting events in a continuous interval (time, area, volume)
  • Approximation of Binomial when n is large and p is small (λ = np)
🔁
Geometric Distribution
Number of trials until first success
DISCRETE

Models how long you wait until the first success. The only discrete distribution with the memoryless property.

PMF (trials until first success) P(X = k) = (1 − p)k−1 · p   for k = 1, 2, 3, …
Expected Value E[X] = 1 / p
Variance Var(X) = (1 − p) / p²
💡
Memoryless property: P(X > s + t | X > s) = P(X > t). No matter how long you've been waiting, the expected remaining wait is the same. Interviewers love asking about this.

⚠️ Convention warning

Some textbooks define X as the number of failures before the first success (starting at 0). Then E[X] = (1−p)/p. Always clarify which convention your interviewer uses.

📊
Normal (Gaussian) Distribution
The bell curve — parameters μ, σ²
CONTINUOUS

The most important distribution in statistics. Arises from the Central Limit Theorem — sums/averages of many independent random variables converge to Normal regardless of the original distribution.

PDF f(x) = (1 / √(2πσ²)) · exp(−(x−μ)² / 2σ²)
Expected Value E[X] = μ
Variance Var(X) = σ²
💡
68-95-99.7 Rule: ~68% of data within ±1σ, ~95% within ±2σ, ~99.7% within ±3σ. This is by far the most-asked "quick recall" fact in interviews.
⏱️
Exponential Distribution
Time between Poisson events — rate λ
CONTINUOUS

Models the waiting time between consecutive events in a Poisson process. The continuous analog of the Geometric distribution.

PDF f(x) = λ · e−λx   for x ≥ 0
Expected Value E[X] = 1 / λ
Variance Var(X) = 1 / λ²
💡
Memoryless (continuous): P(X > s+t | X > s) = P(X > t). The exponential is the only continuous memoryless distribution. Poisson counts events → Exponential measures time between them.
📈
Log-Normal Distribution
If ln(X) ~ Normal, then X ~ Log-Normal
CONTINUOUS

Arises when a variable is the product of many independent positive factors (vs. Normal which is the sum). Always positive, right-skewed.

PDF f(x) = (1 / (x·σ·√(2π))) · exp(−(ln(x)−μ)² / 2σ²)   for x > 0
Expected Value E[X] = exp(μ + σ²/2)
Variance Var(X) = (eσ² − 1) · e2μ+σ²
💡
Why it matters in tech: Revenue per user, session durations, latency times, and stock prices all follow log-normal. When your data is positive-only and right-skewed, try logging it — if it becomes bell-shaped, it's log-normal.

🔗 Critical Relationships

Interviewers love testing whether you understand how distributions connect:

Binomial → Poisson When n → ∞ and p → 0 with np = λ constant, Bin(n,p) ≈ Poisson(λ)
Poisson ↔ Exponential If # events in time t ~ Poisson(λt), then time between events ~ Exp(λ)
Geometric ↔ Exponential Geometric is the discrete analog of Exponential. Both are memoryless.
Normal ↔ Log-Normal If X ~ N(μ,σ²), then eX ~ LogNormal(μ,σ²). Sums → Normal, Products → LogNormal.
Binomial → Normal When n is large, Bin(n,p) ≈ N(np, np(1−p)) — the CLT in action

Discrete vs Continuous — When to Use Which?

Use Discrete When…
You're counting things: clicks, conversions, errors, users, arrivals, defects. The outcome is a whole number. Ask: "Can I count this on my fingers?"
Use Continuous When…
You're measuring things: time, revenue, latency, weight, distance, temperature. The outcome can take any value in an interval. Ask: "Can this be 3.7?"

Real-World FAANG Applications

How these distributions are actually used at top tech companies.

🎯
Binomial in the Wild
Meta A/B Testing
"Out of 10,000 users shown the new News Feed design, how many will click on an ad?" Each user is a Bernoulli trial (click or not), and total clicks ~ Binomial(n=10000, p). This is the foundation of every A/B test significance calculation.
Google Quality Raters
"Out of 100 search results reviewed by human raters, how many are rated 'relevant'?" Binomial models the quality score distribution used to evaluate search ranking algorithms.
Airbnb Booking Conversion
"Of 500 users who view a listing, how many will book?" Conversion funnel analysis uses binomial modeling to set expectations and detect anomalies.
⚡
Poisson in the Wild
Amazon Server Requests
"How many requests will hit our servers per second during Prime Day?" Request counts per time interval follow Poisson — this drives auto-scaling infrastructure decisions.
Netflix Error Rate Monitoring
"How many streaming errors occur per hour?" If suddenly errors spike above the Poisson-expected range, it triggers automated alerts. Anomaly detection at scale.
Apple App Store Reviews
"How many 1-star reviews does an app get per day?" Low-frequency event monitoring uses Poisson to separate normal fluctuation from real quality issues.
🔁
Geometric in the Wild
Google Ad Click-Through
"How many search results does a user scroll through before clicking one?" The position of first click follows a Geometric distribution — informs ad placement and pricing strategy.
Airbnb Search to Book
"How many listings does a guest browse before making their first booking?" Geometric models the trial-until-success pattern in user search behavior.
📊
Normal in the Wild
Meta A/B Test Statistics
By CLT, the average engagement metric across millions of users is approximately Normal. This justifies using z-tests and t-tests for experiment analysis — the backbone of Meta's experimentation platform.
Netflix Recommendation Scores
Predicted rating scores for content tend toward Normal after aggregation. Outliers (very high scores) become recommendations. Understanding the distribution helps set thresholds.
⏱️
Exponential in the Wild
Amazon Time Between Purchases
"How long until a customer's next purchase?" Exponential models inter-arrival times. Drives re-engagement email timing — if a user exceeds the expected wait time, they might be churning.
Google Server Failure Intervals
Time between hardware failures in data centers follows Exponential. Drives capacity planning: how many spare servers do we need to maintain 99.99% uptime?
📈
Log-Normal in the Wild
Amazon Revenue Per Customer
Customer spend is classic log-normal: most spend a little, a few whales spend a lot. Using mean revenue is misleading — median or geometric mean is more representative. Critical for LTV models.
Airbnb Listing Prices & Session Duration
Nightly prices and time-on-site both follow log-normal. Right-skewed, positive-only. Airbnb uses log transforms before modeling to normalize these distributions.
Netflix Content Watch Time
Watch time per session: most sessions are short, some are binges. Log-normal captures this skew. Used for content recommendation and bandwidth planning.

Interview Questions

Conceptual Questions

When would you use Poisson vs. Binomial? EASY ▼
Binomial when you have a fixed number of trials n and want to count successes. Poisson when you're counting events in a continuous interval (time/space) with no fixed upper bound. Practically: if you can say "out of n, how many…" → Binomial. If you ask "how many per hour/day…" → Poisson. Also, Poisson approximates Binomial when n is large and p is small (rare events).

Example: "Out of 1000 emails sent, how many bounced?" → Binomial.
"How many support tickets come in per hour?" → Poisson.
What is the memoryless property? Which distributions have it? EASY ▼
Memoryless: P(X > s+t | X > s) = P(X > t). The probability of having to wait t more units doesn't depend on how long you've already waited s.

Two distributions have it: Geometric (discrete) and Exponential (continuous). They are the only distributions with this property in their respective domains.

Example: If a server's time-to-failure is exponential with mean 100 hours, and it's been running for 80 hours, the expected remaining time is still 100 hours — not 20.
A metric at your company has mean 50 and variance 50. What distribution might this follow? MEDIUM ▼
Poisson with λ = 50. The defining property of Poisson is that E[X] = Var(X) = λ. When you observe mean ≈ variance for count data, Poisson is the first candidate.

Follow-up trap: If the variance is much larger than the mean (overdispersion), the data may follow a Negative Binomial instead. This is extremely common in real-world count data (e.g., page views per user).
Why do we take log transforms of revenue data? What distribution does this suggest? MEDIUM ▼
Revenue data is typically right-skewed and positive-only → classic Log-Normal. If X ~ LogNormal, then log(X) ~ Normal. Log-transforming makes the data approximately Normal, which lets us:

1. Use parametric tests (t-tests, ANOVA) that assume normality
2. Get more stable means (less influenced by whales/outliers)
3. Build better regression models (homoscedastic residuals)

Important nuance: After log-transforming and computing the mean, exponentiating back gives the geometric mean, not the arithmetic mean. The geometric mean is more robust for skewed data.
Users arrive at a website at a rate of 3 per minute. What's the probability no one arrives in a 30-second window? MEDIUM ▼
Arrivals follow a Poisson process with rate λ = 3/min. For a 30-second window, λ30s = 3 × 0.5 = 1.5.

P(X = 0) = e−1.5 · 1.50 / 0! = e−1.5 ≈ 0.223

About a 22.3% chance of no arrivals in 30 seconds.

Equivalently (Exponential view): Time until first arrival ~ Exp(3/min). P(T > 0.5 min) = e−3(0.5) = e−1.5 ≈ 0.223. Same answer — showing the Poisson-Exponential duality.

Scenario-Based (FAANG-Style)

You run an A/B test with 5000 users per group and 2% baseline conversion. How would you model the number of conversions? MEDIUM ▼
Exact: Conversions ~ Binomial(n=5000, p=0.02), so E[X] = 100, Var(X) = 98.

Approximation 1 (Poisson): Since n is large and p is small, Poisson(λ=100) works well.

Approximation 2 (Normal): Since np = 100 >> 10, the Normal approximation N(100, 98) is excellent. This is what we actually use in practice to compute z-scores and p-values.

The full answer connects all three distributions — showing you understand when each approximation is valid is what sets apart a great answer.
Amazon notices that time between server failures follows an exponential distribution with λ = 0.1 failures/hour. What's the probability the server survives at least 24 hours? HARD ▼
Time to failure T ~ Exp(λ = 0.1).

P(T > 24) = e−0.1 × 24 = e−2.4 ≈ 0.0907 ≈ 9.1%

Only about 9% chance the server survives a full day without failure.

Follow-up: "If you have 10 independent servers, what's the probability at least one fails in 24 hours?"
P(at least one fails) = 1 − P(all survive) = 1 − (0.0907)10 ≈ 1 − essentially 0 ≈ ~100%. Wait — actually P(all survive) = (0.0907)10 is near zero, so P(at least one fails) ≈ 1. But that means P(each survives) = 0.0907, and for ALL to survive: (0.0907)10 → almost certainly at least one will fail. This motivates redundancy!
An Airbnb DS notices that booking amounts have a mean of $200 but the median is $120. What distribution might this be? How would you analyze it? HARD ▼
Mean >> Median is the hallmark of a right-skewed distribution → likely Log-Normal.

Approach:
1. Plot histogram → confirm right skew
2. Plot log(amount) histogram → should look Normal
3. Use QQ plot of log(amount) against Normal quantiles
4. Fit log-normal parameters: μ, σ from the logged data

Practical implications:
• Report median ($120) not mean ($200) as the "typical" booking
• Use geometric mean for aggregations
• Log-transform before regression modeling
• For A/B tests on revenue: consider testing log(revenue) or use non-parametric tests, since the raw distribution violates normality assumptions for small samples
You flip a biased coin (P(H) = 0.3) until you get heads. On average how many flips? If you've already flipped 5 tails, what's the expected number of remaining flips? MEDIUM ▼
Geometric with p = 0.3. E[X] = 1/p = 1/0.3 ≈ 3.33 flips.

After 5 tails? Still 3.33 flips expected remaining. This is the memoryless property! P(X > s+t | X > s) = P(X > t). The past failures don't matter.

Why interviewers ask this: It tests whether you truly understand memorylessness vs. just memorizing the formula. Many candidates intuitively feel it should take fewer flips since "we're due for a heads" — that's the gambler's fallacy.
